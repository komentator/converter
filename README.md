# SV Converter - COMTRADE → IEC 61850-9-2 LE

Программа для конвертации осциллограмм в формате COMTRADE (.cfg/.dat) в поток Sampled Values (SV) по стандарту IEC 61850-9-2 LE с упаковкой в PCAP-файл.

## Основные возможности

✅ Парсинг файлов COMTRADE (.cfg и .dat в ASCII-формате)
✅ Автоматическое определение соответствия каналов SV потоку (Ia, Ib, Ic, In, Ua, Ub, Uc, Un)
✅ **НОВОЕ:** Автоподтягивание пары файлов (.cfg + .dat с одинаковым именем)
✅ Ручное редактирование сопоставления каналов
✅ Конфигурирование параметров SV потока:
  - MAC-адрес (Destination MAC)
  - APPID (в шестнадцатиричной форме)
  - VLANID (в шестнадцатиричной форме)
  - VLAN Priority (PCP)
  - svID (идентификатор потока)
  - confRev (версия конфигурации)
  - Флаг Simulation/Test
✅ Задание коэффициентов трансформации:
  - Ктт (трансформация тока)
  - Ктн (трансформация напряжения)
  - K3i0 (коэффициент нейтрального тока)
  - K3u0 (коэффициент нейтрального напряжения)
✅ Генерация корректного PCAP файла со 100% соответствием стандарту IEC 61850-9-2 LE
✅ **НОВОЕ:** Встроенный просмотр данных с графиками и анализом:
  - Осциллограммы токов и напряжений с интерактивным зумом
  - Векторные диаграммы (фазорное представление)
  - Анализ частоты сигнала
  - Табличное представление данных

## Интерфейс программы

### Шаг 1: Загрузка файлов
1. Нажмите на поле "Файл .cfg" и выберите конфигурационный файл осциллограммы
2. **АВТОМАТИЧЕСКИ:** Программа найдет и загрузит .dat файл с таким же именем (если существует)
3. Или наоборот: выберите .dat файл, и .cfg загрузится автоматически

### Шаг 2: Сопоставление каналов
После загрузки файлов программа автоматически попытается определить соответствие каналов.
Вы можете:
- Использовать автоматическое определение кнопкой "🔍 Автоматическое определение"
- Вручную выбрать из выпадающих списков нужные каналы для каждого параметра

### Шаг 3: Параметры потока SV
Заполните необходимые параметры:

**Сетевые параметры:**
- **Destination MAC**: MAC-адрес целевого устройства (формат: 01-0C-CD-04-00-01)
- **APPID**: ID приложения в шестнадцатиричном формате (по умолчанию: 4000)
- **VLANID**: ID виртуальной сети в hex (0 = без VLAN)
- **VLAN Priority (PCP)**: Приоритет (по умолчанию 4, как рекомендует IEC 61850)

**SV ASDU параметры:**
- **svID**: Идентификатор потока Sampled Values
- **confRev**: Номер версии конфигурации (увеличивается при изменении конфига)
- **Флаг Simulation/Test**: Установите, если это тестовые данные

**Коэффициенты трансформации:**
- **Ктт**: Коэффициент трансформации токов (первичное / вторичное)
- **Ктн**: Коэффициент трансформации напряжений
- **K3i0**: Коэффициент для нейтрального тока
- **K3u0**: Коэффициент для нейтрального напряжения

### Шаг 4: Конвертация и сохранение
1. Нажмите кнопку "▶ Конвертировать в PCAP"
2. После успешной конвертации:
   - Нажмите "⬇ Сохранить PCAP" для сохранения файла
   - Или нажмите "📊 Просмотр данных" для анализа

### Шаг 5: Просмотр данных (НОВОЕ!)
Вкладка с мощными инструментами анализа:

**Режим "Осциллограммы":**
- Графики токов (Ia, Ib, Ic, In) и напряжений (Ua, Ub, Uc, Un)
- Интерактивный зум — выделите область мышью для увеличения
- Панорамирование — перетаскивайте график
- Сохранение графиков в PNG/SVG
- Инструменты matplotlib: Home, Pan, Zoom, Save

**Режим "Векторная диаграмма":**
- Фазорное представление всех каналов
- Отображение действующих значений (RMS)
- Визуализация фазовых углов
- Цветовое кодирование: токи (сплошные), напряжения (пунктир)

**Режим "Частота":**
- Расчет мгновенной частоты по нулевым переходам
- График изменения частоты во времени
- Сравнение с номинальной частотой (50/60 Гц)
- Отображение средней частоты

**Режим "Таблица значений":**
- Все 8 каналов в табличном виде
- Прокрутка для просмотра всех сэмплов
- Статистика: количество отсчетов, частота дискретизации, длительность

По умолчанию программа использует следующие параметры (соответствуют примеру в документации):

```
APPID: 4000 (hex)
VLANID: 0 (без VLAN)
svID: RET61850_SV1
confRev: 1
Simulation: включен
Ктт: 1000
Ктн: 1100
K3i0: 1000
K3u0: 1905.2
```

## Технические детали

### Масштабирующие коэффициенты (IEC 61850-9-2 LE)
- **Токи**: 1 LSB = 1 мА (значение × 1000)
- **Напряжения**: 1 LSB = 10 мВ (значение × 100)

### Порядок каналов в SV потоке
Независимо от порядка в исходном файле .cfg, программа всегда генерирует SV поток в стандартном порядке:
```
Ia, Ib, Ic, In, Ua, Ub, Uc, Un
```

### Структура PCAP
- Стандартный формат libpcap (совместим с Wireshark)
- Каждый образец — один кадр Ethernet со структурой SV ASDU по IEC 61850
- Временная метка рассчитывается на основе частоты дискретизации

## Архитектура

- **converter.py**: Основная логика конвертации (парсинг COMTRADE, генерация PCAP)
- **gui_main.py**: GUI приложение на Tkinter с 5 вкладками:
  - Загрузка файлов (с автоподтягиванием пары)
  - Сопоставление каналов
  - Параметры SV
  - Конвертация и сохранение
  - Просмотр данных (графики, векторные диаграммы, анализ)
- **main.py**: Точка входа (запуск GUI)
- **server.py**: Встроенный HTTP-сервер (веб-версия, опционально)
- **sv_converter.spec**: Конфигурация PyInstaller

## Проверка результата

Сгенерированный PCAP файл можно открыть в:
- **Wireshark** — для анализа кадров SV потока
- **IEC 61850 анализаторами** (например, OMICRON CMC, SEL LogicTester)
- Любыми приложениями, поддерживающими формат libpcap

## Известные ограничения

- Поддержка только ASCII формата .dat файлов (не Binary, BINARY32)
- Одиночные ASDU в кадре (как в IEC 61850-9-2 LE Light Edition)
- 80 отсчётов за период (фиксировано по стандарту)
- Поддержка только 8 каналов (4 тока + 4 напряжения) — стандартная конфигурация 9-2 LE

## Поддержка

При возникновении вопросов проверьте:
1. Формат .cfg файла (должен быть COMTRADE 1999 или новее)
2. Формат .dat файла (ASCII, с разделителем-запятой)
3. Логи в консоли программы при её запуске
4. Корректность выбранного соответствия каналов
