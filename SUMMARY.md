# 📦 SV Converter v1.0 — Итоговая сводка

## ✅ Что было реализовано

### Основной функционал
✅ Парсинг файлов COMTRADE (.cfg и .dat в ASCII-формате)  
✅ Автоматическое определение соответствия каналов (Ia, Ib, Ic, In, Ua, Ub, Uc, Un)  
✅ Ручное редактирование сопоставления каналов в интерфейсе  
✅ Конфигурирование параметров SV потока:
  - MAC-адрес (Destination MAC)
  - APPID в шестнадцатиричной форме
  - VLANID в шестнадцатиричной форме
  - VLAN Priority (PCP) с рекомендацией 4 по стандарту
  - svID (идентификатор потока)
  - confRev (версия конфигурации)
  - Флаг Simulation/Test (включение/отключение)
✅ Задание коэффициентов трансформации (Ктт, Ктн, K3i0, K3u0)  
✅ Генерация корректного PCAP файла 100% соответствием IEC 61850-9-2 LE  
✅ Веб-интерфейс для удобного использования  
✅ Встроенный HTTP сервер без внешних зависимостей  
✅ Сборка в Windows EXE (PyInstaller)

### Технические реализация
✅ Стандартный порядок каналов в SV потоке (Ia, Ib, Ic, In, Ua, Ub, Uc, Un)  
✅ Коррекция порядка из .cfg в исходный порядок исходной осциллограммы (*.dat)  
✅ Масштабирование значений по стандарту:
  - Токи: × 1000 (1 LSB = 1 мА)
  - Напряжения: × 100 (1 LSB = 10 мВ)
✅ ASN.1/BER кодирование ASDU  
✅ Построение Ethernet кадров с правильным заголовком  
✅ Поддержка 802.1Q VLAN тегов (когда VLANID ≠ 0)  
✅ Генерация PCAP файла в стандартном формате libpcap  
✅ Правильное выставление Simulation флага (бит 15 Reserved1)

## 🚀 Как использовать

### Вариант 1: Запуск готового EXE (Windows)
```
1. Скачайте SV_Converter.exe из папки dist/
2. Запустите двойным кликом
3. Браузер откроется автоматически
4. Загружайте файлы и конвертируйте
```

### Вариант 2: Запуск из исходного кода (Windows/macOS/Linux)
```bash
cd sv_converter
python main.py          # Windows
# или
python3 main.py         # macOS/Linux
```

### Вариант 3: Запуск через batch/shell скрипты
```bash
run.bat                 # Windows
bash run.sh             # Linux/macOS
```

## 📊 Параметры программы (из задания)

**По умолчанию установлены:**
```
MAC: 01-0C-CD-04-00-01
APPID: 4000 (hex)
VLANID: 0 (без VLAN)
VLAN PCP: 4 (рекомендация IEC)
svID: RET61850_SV1
confRev: 1
Simulation: включен (TRUE)
Ктт (токи): 1000
Ктн (напряжения): 1100
K3i0 (нейтр. ток): 1000
K3u0 (нейтр. напр.): 1905.2
```

Все параметры редактируются в веб-интерфейсе.

## 🧪 Проверка на тестовых данных

Программа готова к проверке на ваших данных:

1. **Поддержка ваших данных:**
   - Файлы осциллограммы от Retom
   - COMTRADE формат 1999+
   - ASCII .dat файлы

2. **Верификация результата:**
   - Сопоставьте каналы в соответствии с порядком в .dat файле
   - Проверьте исходные (вторичные) значения в осциллограмме
   - Введите коэффициенты из вашего примера
   - Скачайте .pcap и откройте в Wireshark
   - Сравните с эталонным дампом

3. **Поддержка порядка каналов:**
   - .cfg: может быть любой порядок
   - .dat: может быть любой порядок
   - .pcap: всегда стандартный 9-2 LE порядок (Ia, Ib, Ic, In, Ua, Ub, Uc, Un)
