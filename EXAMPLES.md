# Примеры использования SV Converter

## Пример 1: Базовая конвертация (как в задании)

Если у вас есть файлы осциллограммы (например, от Retom):

### Параметры конвертации:
```
Файл .cfg: oscillogram.cfg
Файл .dat: oscillogram.dat

MAC: 01-0C-CD-04-00-01
APPID: 4000 (в hex)
VLANID: 0
svID: RET61850_SV1
confRev: 1
Simulation/Test: ✓ включено
Ктт: 1000
Ктн: 1100
K3i0: 1000
K3u0: 1905.2
```

### Ожидаемый результат:

Если в исходной осциллограмме были значения (на первом отсчёте):
```
Ia = 1 A, 0°
Ib = 1 A, 240°
Ic = 1 A, 120°
In = 3 A, 180°
Ua = 57 V, 10°
Ub = 57 V, 250°
Uc = 57 V, 130°
Un = 100 V, 190°
```

То в SV потоке (PCAP) они будут преобразованы в:
```
Ia = 1000 A, 0°
Ib = 1000 A, 240°
Ic = 1000 A, 120°
In = 3000 A, 180°
Ua = 62700 V (57 × 1100), 10°
Ub = 62700 V (57 × 1100), 250°
Uc = 62700 V (57 × 1100), 130°
Un = 190520 V (100 × 1905.2), 190°
```

### Этапы в программе:

1. **Загрузка файлов**
   - Нажимаете на "Файл .cfg" → выбираете `oscillogram.cfg`
   - Нажимаете на "Файл .dat" → выбираете `oscillogram.dat`

2. **Автоматическое сопоставление**
   - После загрузки нажимаете "🔍 Автоматическое определение"
   - Программа подберёт каналы по названиям

3. **Проверка сопоставления**
   ```
   Ia → Channel 2 (Ia, a, A)
   Ib → Channel 3 (Ib, b, A)
   Ic → Channel 4 (Ic, c, A)
   In → Channel 5 (In, n, A)
   Ua → Channel 6 (Ua, a, V)
   Ub → Channel 7 (Ub, b, V)
   Uc → Channel 8 (Uc, c, V)
   Un → Channel 1 (Un, n, V)
   ```
   
   Если порядок не совпадает, отредактируйте вручную.

4. **Заполнение параметров SV**
   - Все уже заполнены дефолтными значениями
   - Вы можете их менять в соответствии с требованиями

5. **Конвертация**
   - Нажимаете "▶ Конвертировать в PCAP"
   - Ждёте сообщения "✅ Успешно!"
   - Нажимаете "⬇ Скачать файл .pcap"

## Пример 2: Специализированная конфигурация

Если вам нужна другая конфигурация (например, для иного оборудования):

### Параметры:
```
MAC: 02-3D-5E-01-00-05          (другой MAC целевого устройства)
APPID: 8001 (в hex)            (другой ID приложения)
VLANID: 100 (в hex)            (включить VLAN)
VLAN Priority: 4               (рекомендуемый приоритет для SV)
svID: TestMU_SV2               (собственный идентификатор)
confRev: 5                     (версия конфигурации)
Simulation/Test: ✓ включено
Ктт: 800                       (другой коэффициент трансформации)
Ктн: 1200
K3i0: 800
K3u0: 2000
```

### Результат:

Значения будут пересчитаны по новым коэффициентам:
- Если в осциллограмме Ia = 1 A, то в SV потоке Ia = 1 × 800 = 800 A
- Если в осциллограмме Ua = 57 V, то в SV потоке Ua = 57 × 1200 = 68400 V

## Пример 3: Проверка результата в Wireshark

После скачивания `sampled_values.pcap`:

1. Откройте Wireshark
2. File → Open → выберите `sampled_values.pcap`
3. В фильтре введите: `eth.dst == 01:0c:cd:04:00:01`
4. Вы увидите кадры SV потока с:
   - Destination MAC: 01:0c:cd:04:00:01
   - EtherType: 0x88ba (Sampled Values)
   - APPID: 0x4000
   - Reserved1: 0x8000 (флаг Simulation=1)

5. Для просмотра значений:
   - Разверните пакет → Sampled Values
   - Посмотрите значения каналов (в байтах, int32 формат, big-endian)

## Пример 4: Интеграция с IEC 61850 устройством

Если вам нужно отправить PCAP поток на реальное устройство:

1. Убедитесь, что MAC адрес соответствует целевому устройству
2. Убедитесь, что APPID согласован с устройством
3. Если используется VLAN, проверьте правильность VLANID
4. Отключите флаг Simulation/Test перед отправкой на реальное оборудование

Пример конфигурации для SEL-401 Merging Unit:
```
MAC: 02-AA-BB-AA-BB-CC          (MAC SEL-401)
APPID: 4000
VLANID: 200                     (используется в сети)
svID: SEL401_MU_SV1
confRev: 1
Simulation: ✗ отключено        (реальные данные)
Ктт: 1000                       (зависит от ТТ)
Ктн: 1100                       (зависит от ТН)
K3i0: 1000
K3u0: 1905.2
```

## Пример 6: Пакетная обработка нескольких файлов

Если нужно конвертировать несколько осциллограмм с одинаковыми параметрами:

1. Заполните все параметры один раз
2. Загрузите первую пару файлов (cfg + dat)
3. Нажимаете "Конвертировать" и скачиваете результат
4. Загрузите вторую пару файлов
5. Все параметры остаются заполненными
6. Нажимаете "Конвертировать" снова
7. Скачиваете второй результат

Параметры сохраняются в браузере между сеансами.

## Вычисление коэффициентов трансформации

### Для трансформаторов тока (ТТ):
```
Ктт = Первичный номинальный ток / Вторичный номинальный ток

Например:
- ТТ 1000/5 → Ктт = 1000 / 5 = 200
- ТТ 600/1 → Ктт = 600 / 1 = 600
- ТТ 400/5 → Ктт = 400 / 5 = 80
```

### Для трансформаторов напряжения (ТН):
```
Ктн = Первичное номинальное напряжение / Вторичное номинальное напряжение

Например:
- ТН 110000/100 → Ктн = 110000 / 100 = 1100
- ТН 35000/100 → Ктн = 35000 / 100 = 350
- ТН 6000/100 → Ктн = 6000 / 100 = 60
```

### Для нейтральных каналов:
```
K3i0 = обычно равен Ктт (если у нас есть ТТ нейтрали с тем же коэффициентом)
K3u0 = обычно равен Ктн, но может быть иным, зависит от схемы подключения

В нашем примере:
- K3i0 = 1000 (такой же как Ктт)
- K3u0 = 1905.2 (может быть иным, задаётся по схеме)
```

## Трубка отладки (Debug)

Если конвертация не работает, проверьте консоль браузера (F12):

1. Откройте Developer Tools (F12)
2. Перейдите на вкладку "Console"
3. Повторите конвертацию
4. Посмотрите сообщения об ошибках
5. Проверьте, что все параметры заполнены корректно

### Типичные ошибки:

**Error: "Не назначен канал для роли Ia"**
- Решение: Убедитесь, что в выпадающем списке выбраны все 8 каналов

**Error: "Некорректный MAC-адрес"**
- Решение: Проверьте формат: должен быть XX-XX-XX-XX-XX-XX или XX:XX:XX:XX:XX:XX

**Error: "Некорректный APPID"**
- Решение: APPID должен быть шестнадцатиричным числом (например, 4000, 8001, FFFF)

## Проверка в реальном оборудовании

После конвертации и скачивания PCAP файла:

### Для SEL-401 Merging Unit:
1. Используйте SEL Logic Tester или CMC
2. Загрузите PCAP файл
3. Проверьте, что устройство принимает SV кадры
4. Снимите показания и сравните с исходными значениями

### Для ABB REX-540:
1. Используйте программное обеспечение для тестирования
2. Проверьте APPID и svID в настройках устройства
3. Убедитесь, что MAC адреса совпадают

### Для OMICRON CMC:
1. Откройте COMTRADE VIEW или другое ПО CMC
2. Загрузите PCAP в режиме анализа
3. Проверьте соответствие значений

## Частые вопросы

**Q: Какой порядок каналов в SV потоке?**
A: Всегда стандартный для 9-2 LE: Ia, Ib, Ic, In, Ua, Ub, Uc, Un (независимо от порядка в исходном файле)

**Q: Что такое флаг Simulation?**
A: Когда включен, устройства-подписчики знают, что это тестовые данные, и не используют их для защиты

**Q: Какая размер одного кадра SV?**
A: Примерно 110 байт (зависит от длины svID). В PCAP файле будет много таких кадров

**Q: Могу ли я редактировать PCAP вручную?**
A: Нет, PCAP - бинарный формат. Используйте эту программу для генерации

**Q: Что делать, если коэффициенты неизвестны?**
A: Установите их все в 1.0 - значения не будут масштабироваться
