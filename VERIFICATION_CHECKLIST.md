# ✅ Чек-лист проверки SV Converter

Используйте этот чек-лист, чтобы убедиться, что программа работает корректно на ваших данных.

## 📋 Предварительная подготовка

- [ ] Наготове файлы осциллограммы (.cfg и .dat)
- [ ] Известны значения исходных (вторичных) параметров осциллограммы
- [ ] Известны коэффициенты трансформации (Ктт, Ктн, K3i0, K3u0)
- [ ] Известны требуемые параметры SV потока (MAC, APPID, VLANID, svID и т.д.)

## 🚀 Запуск программы

- [ ] Программа запустилась без ошибок
- [ ] Сервер слушает на http://127.0.0.1:8000
- [ ] Браузер открыт с интерфейсом программы
- [ ] Все элементы интерфейса видны (нет сообщений об ошибках)

## 📁 Загрузка файлов

- [ ] Файл .cfg загружен успешно
  - [ ] Отображено количество каналов
  - [ ] Визуально отображены имена каналов
- [ ] Файл .dat загружен успешно
  - [ ] Видно сообщение об успешной загрузке
- [ ] Оба файла загружены

## 🔀 Сопоставление каналов

- [ ] Таблица с 8 роля (Ia, Ib, Ic, In, Ua, Ub, Uc, Un) видна
- [ ] Нажата кнопка "🔍 Автоматическое определение"
- [ ] Все выпадающие списки заполнены (ни один не пуст)
- [ ] Сопоставление логично:
  - [ ] Ia сопоставлена с каналом, имеющим "a" фазу и "A" единицу
  - [ ] Ib сопоставлена с каналом, имеющим "b" фазу и "A" единицу
  - [ ] ... и т.д. для остальных параметров
  - [ ] Un сопоставлена с каналом, имеющим "n" фазу и "V" единицу

**Если сопоставление неправильное:**
- [ ] Отредактировано вручную через выпадающие списки

## ⚙️ Параметры SV потока

### Сетевые параметры
- [ ] MAC адрес заполнен (формат: 01-0C-CD-04-00-01 или 01:0C:CD:04:00:01)
- [ ] APPID заполнен в hex (например, 4000)
- [ ] VLANID заполнен в hex (0, если без VLAN)
- [ ] VLAN Priority (PCP) заполнен (по умолчанию 4)

### SV ASDU параметры
- [ ] svID заполнен (например, RET61850_SV1)
- [ ] confRev заполнен (например, 1)
- [ ] Флаг Simulation/Test установлен или отключен в зависимости от требования

### Коэффициенты трансформации
- [ ] Ктт заполнен (например, 1000)
- [ ] Ктн заполнен (например, 1100)
- [ ] K3i0 заполнен (например, 1000)
- [ ] K3u0 заполнен (например, 1905.2)

**Проверка значений:**
- [ ] Все значения являются положительными числами
- [ ] Дробные числа используют точку (.) как разделитель

## 🔄 Конвертация

- [ ] Нажата кнопка "▶ Конвертировать в PCAP"
- [ ] Появилось сообщение статуса (не красное сообщение об ошибке)
- [ ] Сообщение показывает:
  - [ ] "✅ Успешно!"
  - [ ] Количество созданных кадров (примерно 4000 для 1 сек)
  - [ ] Размер PCAP файла в KB

**Если ошибка:**
- [ ] Проверены все обязательные поля заполнены
- [ ] Проверены форматы (MAC, hex значения и т.д.)
- [ ] Проверено, что все каналы сопоставлены
- [ ] Посмотрена информация об ошибке в статус-строке

## 📥 Скачивание файла

- [ ] Появилась кнопка "⬇ Скачать файл .pcap"
- [ ] Нажата кнопка скачивания
- [ ] Файл `sampled_values.pcap` скачан в папку Downloads (или назначенную папку)
- [ ] Размер файла разумный (>100 KB для 1 сек записи)

## 🔍 Проверка результата в Wireshark

### Подготовка
- [ ] Установлен Wireshark (https://www.wireshark.org/)
- [ ] Wireshark запущен

### Открытие файла
- [ ] File → Open → выбран скачанный `sampled_values.pcap`
- [ ] Файл открыт без ошибок
- [ ] Видны пакеты в списке (иконки, номера, размеры)

### Проверка структуры
- [ ] Выбран первый пакет (кликом на нём)
- [ ] На вкладке снизу видна дерево пакета:
  - [ ] Frame (размер ~110-120 байт)
  - [ ] Ethernet II
    - [ ] Destination: 01:0c:cd:04:00:01 (ваш MAC)
    - [ ] Source: 10:ff:e0:84:fe:34 (фиксированный)
    - [ ] Type: 0x88ba (SV - Sampled Values)

### Проверка Sampled Values
- [ ] Развёрнут раздел "Sampled Values"
  - [ ] APPID: 0x4000 (ваш APPID в hex)
  - [ ] Length: примерно 0x006e (110 байт)
  - [ ] Reserved1: 0x8000 (если Simulation=TRUE) или 0x0000
  - [ ] Reserved2: 0x0000 (всегда)

### Проверка ASDU
- [ ] Развёрнут раздел ASDU
  - [ ] svID: "RET61850_SV1" (ваш svID, видно в ASCII)
  - [ ] smpCnt: 0 (для первого пакета)
  - [ ] confRev: 0x00000001 (ваш номер версии)
  - [ ] smpSynch: 0x00 (синхронизация)

### Проверка значений каналов
- [ ] Развёрнут раздел Data (8 каналов × 8 байт)
- [ ] Видны значения в hex-формате (4 байта значение + 4 байта quality)

**Пример для первого кадра (из задания):**
```
Ia: 00 00 03 e8 00 00 00 00  (значение: 0x000003e8 = 1000 в decimal)
Ib: ff ff fc 18 00 00 00 00  (значение: отрицательное)
...
```

## 📊 Проверка значений (продвинутая)

Если у вас есть эталонный дампа PCAP:

- [ ] Скачали эталонный файл
- [ ] Открыли его в Wireshark
- [ ] Сравнили байты первого пакета:
  - [ ] MAC адреса совпадают
  - [ ] APPID совпадает
  - [ ] svID совпадает
  - [ ] Значения каналов совпадают (в hex)
  - [ ] Quality флаги совпадают

**Для побайтового сравнения:**
1. Wireshark: Packet List → Packet Bytes (внизу)
2. Скопируйте hex дампа первого пакета
3. Сравните с эталоном (начиная с Destination MAC)

## 🧮 Вычисление и проверка значений вручную

Если нужно самостоятельно проверить расчёты:

### Пример из задания:

**Входные данные (вторичные):**
- Ia = 1 A, 0°

**Коэффициенты:**
- Ктт = 1000
- Масштаб = 1000 (для A)

**Расчёт:**
```
Ia_primary = 1 × 1000 = 1000 A
Ia_int32 = 1000 × 1000 = 1,000,000 decimal
         = 0x000F4240 hex (big-endian)

В PCAP файле:
Первые 4 байта значения Ia = 00 0F 42 40
```

- [ ] Вычислено значение совпадает с PCAP файлом
- [ ] Остальные каналы проверены аналогично

## 🐛 Отладка (если что-то не работает)

### Проблема: Программа не запускается
- [ ] Проверена версия Python (≥3.7): `python --version`
- [ ] Все файлы на месте (converter.py, server.py, main.py, static/index.html)
- [ ] Текущая папка — корень проекта (`cd sv_converter`)

### Проблема: Браузер не открывается
- [ ] Сервер запустился (видно сообщение в консоли)
- [ ] Открыт браузер вручную: http://127.0.0.1:8000/
- [ ] Интерфейс загружен

### Проблема: Файлы не загружаются
- [ ] Файлы имеют правильные расширения (.cfg, .dat)
- [ ] Файлы открыты в текстовом редакторе (проверена структура)
- [ ] Файлы не повреждены

### Проблема: Конвертация не работает
- [ ] Все 8 каналов сопоставлены (выпадающие списки не пусты)
- [ ] Все параметры заполнены
- [ ] MAC адрес в правильном формате
- [ ] APPID и VLANID — валидные hex-значения
- [ ] Посмотрена консоль браузера (F12 → Console) на ошибки

### Проблема: Значения в PCAP не совпадают с эталоном
- [ ] Проверены коэффициенты трансформации
- [ ] Проверена исходная осциллограмма (правильны ли вторичные значения)
- [ ] Проверено сопоставление каналов (правильный ли порядок)
- [ ] Пересчитаны значения вручную (совпадают ли с расчётами)
